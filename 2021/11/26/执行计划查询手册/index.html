执行计划查询手册

Posted on 11月 26 2021 阅读次数：

表关联查询

  • 先从驱动表查询出一批数据
  • 再用这批数据去关联表查询

核心就在于，两个表对应的查询条件都应该有对应的索引

成本计算

场景 数值 描述
查一页数据 1.0 io成本
筛选一条数据 0.2 cpu成本

一条sql里可能用的到的索引叫做possibleKeys

查询一个表有多少页

  1. 执行 show table status like 表名
  2. 获取data_length
  3. 计算 页 = data_length / 16kb
  4. 获取rows
  5. 总成本 = io成本+cpu成本 = 页数 * 1.0 + rows * 0.2

  • 遍历物化表，到主表查索引
